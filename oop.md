먼저 1. 객체지향 프로그래밍이란?
객체지향 프로그래밍은 프로그램을 객체라는 단위로 나누어 설계하고 구현하는 프로그래밍 방식입니다.

여기서 객체는 데이터와 동작을 함꼐 가지고 있으며, 현실세계의 사물을 모데링하여 프로그램을 더 쉽게 이해하고 관리할 수 있도록 하는 것 입니다.

예를 들면 자동차 객체가 있다.

2. 객체지향 프로그래밍의 특징
첫번째로 캡슐화로 데이터와 메서드를 하니의 객체 안에 묶고, 외부에서 직접 접근하지 못하도록 보호하는 것입니다.
두번째는 상속으로 기존 클래스의 속성과 기능을 새로운 클래스가 물려받아서 사용하는 것입니다.
장점으로는 코드 재사용이 가능하고 프로그램 구조가 단순합니다.
마지막 세번쨰로는 추상화이고 필요한 정보만 보여주고 복잡한 내부 구조는 숨기는 것입니다.
에시로는 자동차를 운전하떄 우리는 핸들과 페달만 사용하고 엔진 내부 구조까지 알 필요는 없는 것과 같습니다.'

3.객체지향 프로그래밍의 장점
코드 재사용성이 높고 유지보수가 쉽고 프로그램 구조가 명확해지고 현실 세계를 모델링하기 쉽다는 점이 있습니다.

4.객체지향 프로그래밍의 단점
프로그램이 복잡해질 수 있고 실행 속도가 느릴수 있고 설게 하기가 어렵고 메모리 사용량이 많아질수 있습니다.

마지막 5.객체지향 프로그래밍이 사용되는 곳
대체로 게임 개발에 사용되고 웹 개발,앱 개발,대규모 소프트웨어 개발에서도 사용되고 있고 대표적으로 Jave, C++,Python 같은 언어들이 객체지향 프로그래밍을 지원합니다.
